03 MODULE

Secure Software Development – Computer Science.

Module description will be provided later.

Main gols for the module:

1. Acquire a critical understanding of the concept of abstraction in programming.

2. Develop an understanding of the basic principles of secure development methodologies.

3. Explore how to undertake analysis, program design, software construction and testing required for software development.

4. Demonstrate an understanding of the basic principles of architecture, as well as traditional and contemporary Software Development Life Cycle (SDLC) models, such as TOGAF and Agile.

5. The ability to put into practice the techniques learned in a team environment, demonstrating how to deal with conflicts and how to make compromises, and be able to critically evaluate the effectiveness of the approaches.

6. Utilise the opportunity to reflect on and evaluate your own personal development.

  01 UNIT

Introduction to Secure Software Development

Required Reading & Reflection

During Unit 1, I delved into key principles for creating secure software, analyzing the shortcomings of conventional development methods like Waterfall and the advantages of contemporary approaches such as Agile. The flexibility and iterative nature of Agile make it the preferred choice for secure, dynamic software environments while promoting a collaborative and responsive development process.
I was also introduced to the Unified Modeling Language (UML), an effective tool for designing software systems using standardized diagrams. As part of our group work, I utilized Umbrello, an open-source UML tool, to create a flowchart illustrating steps that lead to a common security vulnerability identified by OWASP. This exercise demonstrated how UML can simplify complex design elements, aiding in the visualization of system interactions and ensuring accurate representation of secure software models.
Unit 1 also emphasized the importance of cultivating a risk-aware culture within organizations. I discovered that while developing secure software is crucial, it's equally vital to foster an environment where developers and stakeholders actively identify and address risks. Establishing this culture is fundamental for maintaining secure development practices throughout the software lifecycle.
Lastly, this unit included the formation of our project team. We set up a WhatsApp Group for daily communication, signed Team Contract, and created a shared Google Drive folder for collaborative work. This initial organization lays the groundwork for a successful team project by ensuring clear communication, shared responsibilities, and easily accessible resources.

UML Modelling to Support Secure System Planning

Required Reading & Reflection

In Unit 2, I developed skills in UML modeling to enhance secure system design within the Agile Software Development Life Cycle (SDLC). This unit emphasized the role of UML diagrams and flowcharts as essential tools for effective communication among stakeholders, beginning in the design phase and supporting development throughout the process.
A key focus of this unit was integrating security practices at each stage of the Agile SDLC. I explored best practices that proactively address security vulnerabilities by embedding protective measures in every phase of development. To build on this, I engaged in hands-on practice with UML diagrams that represent secure system structures and workflows. These included use case diagrams to map user interactions, sequence diagrams detailing step-by-step workflows, activity diagrams highlighting user actions, and class diagrams outlining the relationships, attributes, and methods involved in the system architecture. Each of these modeling techniques helped me visualize how security considerations influence the design and functionality of software systems.
To further understand the standards that guide secure system planning, I studied ISO/IEC guidelines on software security, with a particular focus on the ISO/IEC 27000 series. Familiarity with this industry-standard terminology and framework allowed me to apply consistent security principles in my UML models and to better understand the language that underpins secure software development.

  04 UNIT

Exploring Programming Language Concepts

Required Reading & Reflection

This unit focused on two key programming concepts: regular expressions (RegEx) and recursion, and their implications for system security. The activities and discussions provided valuable insights into the practical applications, benefits, and challenges of these concepts.
I explored how RegEx can serve as a powerful tool for tasks like data validation, parsing, and text search. However, I also examined the security risks it introduces, particularly Regular Expression Denial of Service (ReDOS) attacks caused by problematic patterns known as "Evil Regex." Through these discussions, I learned strategies to mitigate such risks, including simplifying patterns, implementing timeouts, and pre-testing expressions for safety and efficiency.
The concept of recursion, where a function calls itself to solve problems such as sorting, searching, or traversing data structures, was another area of focus. While recursion offers an elegant solution to many programming challenges, I analyzed how improper implementation could lead to risks like stack overflow. Implementing proper safeguards, such as defining clear base cases and considering tail recursion optimization, were identified as critical measures for secure and efficient use.
Understanding the security implications of these concepts was a significant part of this unit. I gained insight into how inefficient or overly complex implementations of RegEx or recursion could lead to vulnerabilities, such as system performance degradation or denial-of-service risks. This knowledge helped me learn to balance security requirements with the functionality provided by these tools.
Collaboration was an integral part of this unit. During our third online team meeting on November 24th, we worked on developing a secure software development design document. The recording of this meeting is available here (a link to record).
This unit allowed me to critically evaluate these programming concepts within the context of security, equipping me with practical knowledge to make informed decisions when designing solutions. Click 'EXPLORE UNIT'S ARTEFACTS'.

Introduction to Operating Systems

Required Reading & Reflection

This week, I explored the fundamental concepts of operating systems (OS) and their critical relationships with programming languages and software security. Operating systems are the backbone of most applications, providing essential functions such as handling user inputs, managing processes, and facilitating interactions with software libraries. The unit covered several key topics, including the types and functions of typical operating systems, the differences between processes, threads, and schedulers, and approaches to enhancing OS security. One of the activities involved creating a simple command-line interface (CLI) shell using Python. This shell could perform basic tasks such as listing directory contents, adding numbers, and displaying available commands. I analyzed the shell for potential security vulnerabilities and proposed improvements, including enhanced input validation to mitigate risks.
Additionally, I worked on developing a RESTful API for a distributed environment. Using Python’s Flask library, I implemented an API to manage user records, allowing for creation, deletion, and modification of user data. Running and testing the API provided insights into how libraries like Flask enable seamless web-based interactions while highlighting the importance of secure API design.
This unit also introduced the concept of ontologies and their relevance to software systems. I explored the definition of an ontology and considered its application to the secure CRUD-functional RESTful API my team is designing for the summative assessment.
By the end of this unit, I had a comprehensive understanding of operating systems, their role in software development, and the associated security considerations.   10 UNIT

From Distributed Computing to Microarchitectures

Required Reading & Reflection

This week provided a comprehensive exploration of modern system architectures, from traditional monolithic deployments to distributed systems, microservices, virtualization, and containers. These architectures represent the evolution of computing systems, reflecting the growing demand for scalability, flexibility, and efficiency. However, with these advancements comes an increased attack surface, necessitating robust security measures to protect sensitive data and system integrity.
I analyzed the strengths and weaknesses of these architectures, focusing on the challenges posed by distributed systems. As systems become more decentralized, ensuring secure communication across distributed components becomes critical. Techniques such as encryption and secure key distribution were examined to address these challenges. Additionally, I studied the unique vulnerabilities of virtualized environments and the methods needed to mitigate specific threats in these contexts.
As part of this unit, I reviewed the article by Schmitz et al. (2016) on faceted data as a method to prevent data leakage. This approach involves creating controlled views of sensitive data to limit exposure, a technique that aligns with the principle of least privilege. I evaluated the pros and cons of faceted data: while it offers a robust method for minimizing data leakage, it requires careful implementation and performance considerations to avoid bottlenecks or excessive complexity.
I also designed a basic outline in Python for implementing a faceted data system, demonstrating how controlled access could be enforced programmatically. This exercise provided practical insights into the challenges and benefits of managing data access in distributed and virtualized environments.
By the end of this unit, I gained a deeper understanding of how system architectures have evolved and the security implications of these changes. I also expanded my knowledge of encryption techniques and data leakage prevention, which are vital in building secure, distributed systems.
